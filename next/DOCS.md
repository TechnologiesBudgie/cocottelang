# Cocotte Language Reference

**Version 0.1.0**

Cocotte is a compiled and interpreted programming language with English-like syntax. It targets Linux, Windows, macOS, and BSD across all common CPU architectures. Programs are written in `.cot` files, managed with `Millet.toml`, and built with a single `cocotte` CLI.

## 1. Installation

### Linux (any architecture)

```sh
curl -fsSL https://cocotte-lang.pages.dev/install.sh | sh
```

This detects your architecture (`x86_64`, `aarch64`, `armv7`, `i686`, `riscv64`) and installs the correct binary to `/usr/local/bin/cocotte`.

### Manual

Download a pre-built binary from [GitHub Releases](https://github.com/TechnologiesBudgie/cocottelang/releases), make it executable, and place it in your `PATH`:

```sh
chmod +x cocotte-linux-x86_64
mv cocotte-linux-x86_64 /usr/local/bin/cocotte
```

### From source

Requires a Rust toolchain (`cargo`).

```sh
git clone https://github.com/TechnologiesBudgie/cocottelang
cd cocottelang
cargo build --release
# Binary: target/release/cocotte
```

### Verify

```sh
cocotte --version
```

## 2. Getting Started

```sh
cocotte init MyProject
cd MyProject
cocotte run
```

Output:

```
Hello, World!
Hello, Cocotte!
```

To run an arbitrary file:

```sh
cocotte run path/to/file.cot
```

To compile to a native binary:

```sh
cocotte build --release
./dist/MyProject
```

## 3. Project Structure

```
MyProject/
├── Millet.toml         # project metadata and dependencies
├── Charlotfile         # task runner (replaces Makefile)
├── src/
│   └── main.cot        # entry point
├── libraries/          # local .cotlib files
├── modules/            # installed .cotmod files
├── tests/              # test files (*_test.cot)
├── dist/               # compiled output (generated by cocotte build)
└── .gitignore
```

The entry point is always `src/main.cot` unless overridden on the command line.

## 4. CLI Reference

### `cocotte init <name>`

Scaffolds a new project in a directory named `<name>`.

```sh
cocotte init MyApp
```

Creates `Millet.toml`, `Charlotfile`, `src/main.cot`, `tests/`, `libraries/`, `modules/`, `dist/`, and a `.gitignore`.

---

### `cocotte run [file]`

Interprets and runs a `.cot` file. Executes immediately; no compilation step.

```sh
cocotte run                      # runs src/main.cot
cocotte run src/other.cot        # runs a specific file
cocotte run --debug src/main.cot # enable debug output
cocotte run --bytecode           # use the bytecode VM instead of tree-walk interpreter
```

| Flag        | Description                                     |
|-------------|-------------------------------------------------|
| `--debug`   | Print interpreter state during execution        |
| `--bytecode`| Use bytecode VM (faster for computation-heavy programs) |

---

### `cocotte build [file]`

Compiles a `.cot` file to a native binary. Output goes to `dist/` by default.

```sh
cocotte build                              # compile for current OS and arch
cocotte build --release                    # optimised build (LTO, strip)
cocotte build --os windows                 # compile for Windows x86_64
cocotte build --os linux --arch aarch64    # compile for Linux ARM64
cocotte build --os linux windows --arch x86_64 aarch64  # four binaries at once
```

| Flag              | Description                                        |
|-------------------|----------------------------------------------------|
| `--os <OS...>`    | Target OS(es): `linux`, `windows`, `macos`, `bsd` |
| `--arch <ARCH...>`| Target arch(es): `x86_64`, `aarch64`, `armv7`, `i686`, `riscv64` |
| `--release`       | Enable optimisations (`opt-level 3`, LTO, strip)  |
| `--symbols`       | Include debug symbols                             |
| `--out <dir>`     | Output directory (default: `dist/`)               |
| `--verbose`       | Print internal build steps                        |

When `--os` and `--arch` are both specified, all combinations are built (cartesian product). See [§16 Cross-Compilation](#16-cross-compilation) for toolchain requirements.

Output binary names follow the pattern `<project>-<os>-<arch>[.exe]`. A build for the current machine with no flags uses the plain project name.

---

### `cocotte add <module|library>`

```sh
cocotte add json                        # add built-in module
cocotte add /path/to/utils.cotlib       # add local library
```

Copies the library into `libraries/` and records the dependency in `Millet.toml`.

---

### `cocotte test [dir]`

Runs all files matching `*_test.cot` under the given directory (default: `tests/`).

```sh
cocotte test
cocotte test tests/math_test.cot   # single test file
cocotte test --verbose
```

A test file is a regular `.cot` file that uses `assert` and `assert_eq`. Any failing assertion exits with a non-zero code and prints the error.

---

### `cocotte exec <task>`

Runs a task defined in the `Charlotfile`.

```sh
cocotte exec build
cocotte exec deploy
cocotte exec list       # list all available tasks
```

---

### Other commands

| Command                    | Description                                    |
|----------------------------|------------------------------------------------|
| `cocotte clean`            | Delete `dist/`, `build/`, and cache files      |
| `cocotte package [--format zip\|tar]` | Archive `dist/` into a distributable   |
| `cocotte repl`             | Start the interactive REPL                     |
| `cocotte disasm <file>`    | Print bytecode disassembly for a source file   |

## 5. Syntax

### 5.1 Comments

```cocotte
# This is a single-line comment
var x = 10  # inline comment
```

There are no multi-line comment delimiters. Use multiple `#` lines.

---

### 5.2 Variables

Variables are declared with `var`. They are dynamically typed and can be reassigned to any type.

```cocotte
var name   = "Alice"
var age    = 30
var active = true
var score  = nil
```

Reassignment does not use `var`:

```cocotte
age = 31
```

`nil` is the explicit absence of a value.

---

### 5.3 Types

| Type      | Example literals                        |
|-----------|-----------------------------------------|
| `number`  | `0`, `3.14`, `-7`, `1000`              |
| `string`  | `"hello"`, `"line1\nline2"`            |
| `bool`    | `true`, `false`                         |
| `nil`     | `nil`                                   |
| `list`    | `[1, 2, 3]`, `["a", "b"]`, `[]`        |
| `map`     | `{"key": "value"}`, `{}`               |
| `func`    | `func(x) return x * 2 end`             |

Use `type_of(v)` to get the type name as a string at runtime.

```cocotte
print type_of(42)         # "number"
print type_of("hi")       # "string"
print type_of([1, 2])     # "list"
print type_of(nil)        # "nil"
```

### 5.4 Operators

#### Arithmetic

| Operator          | Description           | Example                        |
|-------------------|-----------------------|--------------------------------|
| `+`               | Add / concatenate     | `5 + 3` → `8`, `"a" + "b"` → `"ab"` |
| `-`               | Subtract              | `5 - 3` → `2`                 |
| `*`               | Multiply              | `4 * 3` → `12`                |
| `divide A by B`   | Divide                | `divide 10 by 3` → `3.333...` |
| `%`               | Remainder             | `10 % 3` → `1`                |

Division uses the `divide A by B` keyword form to make it unambiguous and readable. For integer division, wrap with `floor()`:

```cocotte
print divide 10 by 3          # 3.333...
print floor(divide 10 by 3)   # 3
```

#### Comparison

| Operator | Description           |
|----------|-----------------------|
| `==`     | Equal                 |
| `!=`     | Not equal             |
| `<`      | Less than             |
| `>`      | Greater than          |
| `<=`     | Less than or equal    |
| `>=`     | Greater than or equal |

#### Logical

| Operator | Description         |
|----------|---------------------|
| `and`    | Logical AND         |
| `or`     | Logical OR          |
| `not`    | Logical NOT (unary) |

```cocotte
print true and false     # false
print true or false      # true
print not true           # false
```

#### String concatenation

`+` coerces numbers and booleans to strings automatically:

```cocotte
print "Score: " + 42      # "Score: 42"
print "Active: " + true   # "Active: true"
```

### 5.5 Control Flow

```cocotte
if condition
    # ...
elif other_condition
    # ...
else
    # ...
end
```

All `if` blocks must be closed with `end`. `elif` and `else` are optional. There is no limit on the number of `elif` branches.

```cocotte
var grade = 78

if grade >= 90
    print "A"
elif grade >= 80
    print "B"
elif grade >= 70
    print "C"
else
    print "F"
end
```

Conditions are any expression that evaluates to a boolean. `nil` and `false` are falsy; everything else is truthy.

---

### 5.6 Loops

#### `while`

Repeats as long as the condition is true.

```cocotte
var i = 0
while i < 5
    print i
    i = i + 1
end
```

#### `for`

Iterates over every element of a list.

```cocotte
for item in ["apple", "banana", "cherry"]
    print item
end
```

#### `for` with `range`

`range(start, end)` produces the list `[start, start+1, ..., end-1]`.

```cocotte
for i in range(1, 6)
    print i    # 1 2 3 4 5
end
```

#### `break` and `continue`

```cocotte
for n in range(1, 100)
    if n > 5
        break        # exit the loop immediately
    end
    print n
end

for n in range(1, 11)
    if n % 2 == 0
        continue     # skip to the next iteration
    end
    print n          # 1 3 5 7 9
end
```

Both `break` and `continue` work inside `while` loops as well.

### 5.7 Functions

#### Declaration

```cocotte
func add(a, b)
    return a + b
end

var result = add(3, 4)    # 7
```

#### No parameters

```cocotte
func say_hello()
    print "Hello!"
end

say_hello()
```

#### Early return

A `return` with no value returns `nil`. Functions that reach the end without a `return` also return `nil`.

```cocotte
func check(x)
    if x < 0
        return
    end
    print "positive: " + x
end
```

#### Functions as values (lambdas)

```cocotte
var double = func(x) return x * 2 end
print double(5)    # 10
```

Lambdas can span multiple lines. The `end` keyword closes them like any other function.

```cocotte
var clamp_positive = func(n)
    if n < 0
        return 0
    end
    return n
end
```

#### Higher-order functions

```cocotte
func apply(f, x)
    return f(x)
end

print apply(func(n) return n * n end, 7)    # 49
```

#### Closures

Functions capture variables from their enclosing scope.

```cocotte
func make_adder(n)
    return func(x) return x + n end
end

var add5 = make_adder(5)
print add5(10)    # 15
print add5(20)    # 25
```

#### Recursion

```cocotte
func fib(n)
    if n <= 1
        return n
    end
    return fib(n - 1) + fib(n - 2)
end

print fib(10)    # 55
```

### 5.8 Classes

A class is declared with `class`, has an `init` constructor, and uses `self` to access instance data.

```cocotte
class Point
    func init(x, y)
        self.x = x
        self.y = y
    end

    func distance_to(other)
        var dx = self.x - other.x
        var dy = self.y - other.y
        return sqrt(dx * dx + dy * dy)
    end

    func to_string()
        return "(" + self.x + ", " + self.y + ")"
    end
end

var a = Point(0, 0)
var b = Point(3, 4)
print a.to_string()          # (0, 0)
print a.distance_to(b)       # 5
```

- `init` is called automatically when the object is created.
- All fields are created by assigning to `self.field_name` inside any method.
- There is no access control — all fields and methods are public.
- Cocotte does not have inheritance in 0.1.0. Use composition instead.

### 5.9 Error Handling

```cocotte
try
    var result = divide 10 by 0
catch err
    print "Caught: " + err
end
```

The `catch` variable holds the error message as a string. The block after `end` always runs after either the `try` body or the `catch` body completes.

```cocotte
func safe_read(path)
    try
        return read_file(path)
    catch err
        print "Could not read '" + path + "': " + err
        return nil
    end
end
```

To raise an error explicitly, use `assert`:

```cocotte
assert(age >= 0, "age must be non-negative")
```

## 7. String Methods

Methods are called with a dot: `"hello".upper()`.

| Method                       | Description                                           |
|------------------------------|-------------------------------------------------------|
| `.len()`                     | Number of characters                                  |
| `.is_empty()`                | `true` if the string has zero characters              |
| `.upper()`                   | Uppercase copy                                        |
| `.lower()`                   | Lowercase copy                                        |
| `.trim()`                    | Remove leading and trailing whitespace                |
| `.trim_left()`               | Remove leading whitespace                             |
| `.trim_right()`              | Remove trailing whitespace                            |
| `.get(i)`                    | Character at index `i` (0-based)                      |
| `.slice(from, to)`           | Substring from `from` (inclusive) to `to` (exclusive)|
| `.index_of(sub)`             | First position of `sub`; `-1` if not found            |
| `.contains(sub)`             | `true` if `sub` appears in the string                 |
| `.starts_with(prefix)`       | `true` if the string begins with `prefix`             |
| `.ends_with(suffix)`         | `true` if the string ends with `suffix`               |
| `.replace(from, to)`         | Replace all occurrences of `from` with `to`           |
| `.replace_first(from, to)`   | Replace only the first occurrence                     |
| `.split(sep)`                | Split into a list by separator                        |
| `.split_lines()`             | Split on `\n`; returns a list of lines                |
| `.join(sep)`                 | *(called on lists)* Join elements with separator      |
| `.repeat(n)`                 | Repeat the string `n` times                           |
| `.pad_left(n, char)`         | Pad on the left to total length `n`                   |
| `.pad_right(n, char)`        | Pad on the right to total length `n`                  |
| `.to_number()`               | Parse as a number                                     |
| `.to_list()`                 | Convert to a list of single-character strings         |

```cocotte
var s = "  Hello, World!  "
print s.trim()                         # "Hello, World!"
print s.trim().slice(7, 12)            # "World"
print "2025".pad_left(8, "0")          # "00002025"
print "a,b,c".split(",").join(" - ")   # "a - b - c"
```

## 8. List Methods

| Method                     | Description                                                   |
|----------------------------|---------------------------------------------------------------|
| `.len()`                   | Number of items                                               |
| `.is_empty()`              | `true` if the list has no items                               |
| `.get(i)`                  | Item at index `i` (0-based)                                   |
| `.first()`                 | First item                                                    |
| `.last()`                  | Last item                                                     |
| `.push(val)`               | Append `val` to the end (mutates in place)                    |
| `.pop()`                   | Remove and return the last item (mutates in place)            |
| `.contains(val)`           | `true` if `val` is in the list                                |
| `.index_of(val)`           | Index of `val`; `-1` if not found                             |
| `.slice(from, to)`         | Sub-list from `from` (inclusive) to `to` (exclusive)          |
| `.find(func)`              | First item for which `func` returns `true`; `nil` if none     |
| `.filter(func)`            | New list keeping items where `func` returns `true`            |
| `.map(func)`               | New list with `func` applied to every item                    |
| `.reduce(func, init)`      | Fold all items into one value, starting from `init`           |
| `.each(func)`              | Call `func` on every item; returns `nil`                      |
| `.count(func)`             | Number of items for which `func` returns `true`               |
| `.sort()`                  | Sort in place (numbers: ascending; strings: lexicographic)    |
| `.reverse()`               | Reverse in place                                              |
| `.join(sep)`               | Concatenate all items as strings with `sep` between them      |
| `.extend(other)`           | Append all items from `other` list (mutates in place)         |
| `.copy()`                  | Return a shallow copy                                         |
| `.clear()`                 | Remove all items (mutates in place)                           |

```cocotte
var nums = [5, 3, 8, 1, 9, 2]
nums.sort()
print nums.join(", ")          # 1, 2, 3, 5, 8, 9

var evens = nums.filter(func(n) return n % 2 == 0 end)
print evens                    # [2, 8]

var sum = nums.reduce(func(acc, n) return acc + n end, 0)
print sum                      # 28
```

## 9. Map Methods

| Method           | Description                                           |
|------------------|-------------------------------------------------------|
| `.get(key)`      | Value for `key`; `nil` if the key does not exist      |
| `.set(key, val)` | Set `key` to `val` (creates the key if absent)        |
| `.has_key(key)`  | `true` if `key` exists in the map                     |
| `.keys()`        | List of all keys (insertion order)                    |
| `.values()`      | List of all values (insertion order)                  |

```cocotte

config.set("debug", true)

for key in config.keys()
    print key + " -> " + config.get(key)
end
```

## 10. File I/O

All file functions are built in. No module is required.

```cocotte
# Write (creates or overwrites)
write_file("notes.txt", "First line\n")

# Append
append_file("notes.txt", "Second line\n")

# Read
var content = read_file("notes.txt")
print content

# Process line by line
for line in content.split_lines()
    if not line.trim().is_empty()
        print ">" + line
    end
end
```

### Directory operations

```cocotte
make_dir("data/cache")          # creates all intermediate dirs
var files = list_dir("data")    # list of filenames (not full paths)
for name in files
    print name
end
```

### File system checks

```cocotte
print file_exists("notes.txt")  # true
print is_file("notes.txt")      # true
print is_dir("data")            # true
print file_size("notes.txt")    # size in bytes (number)
```

### Copy, rename, delete

```cocotte
copy_file("notes.txt", "notes.bak")
rename_file("notes.bak", "backup/notes.txt")
delete_file("backup/notes.txt")
delete_file("backup")           # also deletes directories
```

### Full file function reference

| Function                   | Description                              |
|----------------------------|------------------------------------------|
| `read_file(path)`          | Return file contents as a string         |
| `write_file(path, text)`   | Write `text` to `path` (overwrites)      |
| `append_file(path, text)`  | Append `text` to `path`                  |
| `delete_file(path)`        | Delete file or directory                 |
| `file_exists(path)`        | `true` if `path` exists                  |
| `is_file(path)`            | `true` if `path` is a regular file       |
| `is_dir(path)`             | `true` if `path` is a directory          |
| `file_size(path)`          | File size in bytes                       |
| `make_dir(path)`           | Create directory (and all parents)       |
| `list_dir(path)`           | List of filenames in a directory         |
| `copy_file(from, to)`      | Copy a file                              |
| `rename_file(from, to)`    | Move or rename a file                    |

## 11. Modules

Modules add capabilities not present in the core language. Load them with `module add`.

```cocotte
module add "json"
```

The `module add` statement can appear anywhere in a file, but by convention it is placed at the top.

### `math`

```cocotte
module add "math"

print math.PI       # 3.14159265358979...
print math.E        # 2.71828182845904...
print math.TAU      # 6.28318530717958...

print math.sin(math.PI / 2)    # 1
print math.cos(0)              # 1
print math.tan(math.PI / 4)    # ~1
print math.asin(1)
print math.acos(1)
print math.atan(1)
print math.log(math.E)        # 1 (natural log)
print math.log2(8)            # 3
print math.log10(1000)        # 3
print math.exp(1)             # e
print math.sqrt(144)          # 12
```

## 12. Libraries

A library is a `.cotlib` file you write yourself. It can define functions, classes, and constants, and can be shared between multiple projects.

### Creating a library

`libraries/strings_ext.cotlib`:

```cocotte
func title_case(s)
    var words = s.lower().split(" ")
    var result = []
    for word in words
        if word.len() > 0
            result.push(word.get(0).upper() + word.slice(1, word.len()))
        end
    end
    return result.join(" ")
end

func word_count(s)
    return s.trim().split(" ").filter(func(w) return not w.is_empty() end).len()
end
```

### Using a library

```cocotte
library add "libraries/strings_ext.cotlib"

print strings_ext.title_case("hello world")   # Hello World
print strings_ext.word_count("one two three") # 3
```

The module name used to call functions is derived from the filename without the `.cotlib` extension.

### Adding to a project

```sh
cocotte add /path/to/strings_ext.cotlib
```

This copies the file to `libraries/` and records it in `Millet.toml`.

## 13. GUI — Charlotte

Charlotte is Cocotte's GUI module. It is backed by [egui](https://github.com/emilk/egui) and runs on Linux (Wayland and X11), Windows, and macOS.

### Build requirements

Charlotte requires optional Cargo features. Uncomment the lines in `Cargo.toml`:

```toml
eframe = "0.29"
egui   = "0.29"

[features]
default = []
gui = ["eframe", "egui"]
```

Then build the Cocotte binary with:

```sh
cargo build --release --features gui
```

The `cocotte` binary shipped in official releases includes Charlotte by default.

### Opening a window

```cocotte
module add "charlotte"

charlotte.window("App Name", width, height, func(ui)
    # UI code here — called every frame
end)
```

The callback runs approximately 60 times per second. Return values from the callback are ignored.

### Persistent state

Because the callback runs on every frame, local variables inside it are reset each time. Use a `map` declared outside the callback to persist state:

```cocotte
module add "charlotte"

var state = {"count": 0}

charlotte.window("Counter", 400, 300, func(ui)
    ui.label("Count: " + state.get("count"))
    if ui.button("Increment")
        state.set("count", state.get("count") + 1)
    end
end)
```

## 14. Charlotfile — Task Runner

The `Charlotfile` defines named tasks — sequences of shell commands to run together. It replaces `Makefile` for Cocotte projects.

### Syntax

```toml
[project]
name   = "MyApp"
author = "You"

[variables]
OUT_DIR = "dist"
PORT    = "8080"

[tasks.run]
cocotte run

[tasks.build]
cocotte build --release

[tasks.serve]
cd ${OUT_DIR} && python3 -m http.server ${PORT}

[tasks.deploy]
cocotte build --release
scp dist/MyApp user@server:/opt/myapp/
ssh user@server "systemctl restart myapp"

[tasks.clean]
cocotte clean
rm -rf ${OUT_DIR}
```

Each `[tasks.<name>]` section contains one or more shell commands, one per line. Commands run in order; if any command exits with a non-zero code the task stops.

Variable references use the `${VAR}` syntax and are expanded before the command runs.

### Running tasks

```sh
cocotte exec build
cocotte exec deploy
cocotte exec list          # list all defined tasks
```

### Multi-language projects

```toml
[tasks.build_all]
cocotte build --release
cd backend && cargo build --release
cd frontend && npm run build

[tasks.dev]
cocotte run &
cd backend && cargo run &
cd frontend && npm start
```

## 16. Cross-Compilation

`cocotte build` uses Rust's cross-compilation toolchain under the hood. The `--os` and `--arch` flags accept multiple values; all combinations are built.

```sh
# Single target
cocotte build --os windows --arch x86_64

# Multiple OSes, single arch
cocotte build --os linux macos --arch aarch64

# Full matrix: 6 binaries (3 OS × 2 arch)
cocotte build --os linux windows macos --arch x86_64 aarch64
```

### Supported targets

| OS      | x86_64 | aarch64 | armv7 | i686 | riscv64 |
|---------|:------:|:-------:|:-----:|:----:|:-------:|
| Linux   |   ✓    |    ✓    |   ✓   |  ✓   |    ✓    |
| Windows |   ✓    |    ✓    |       |  ✓   |         |
| macOS   |   ✓    |    ✓    |       |      |         |
| BSD     |   ✓    |    ✓    |       |      |         |

Accepted OS strings: `linux`, `windows` (`win`), `macos` (`mac`, `darwin`, `osx`), `bsd` (`freebsd`, `openbsd`, `netbsd`).

Accepted arch strings: `x86_64` (`amd64`, `x64`), `aarch64` (`arm64`), `armv7` (`arm`), `i686` (`i386`, `x86`), `riscv64`.

### Toolchain requirements

To compile for a target different from the host, install the corresponding Rust target and cross-linker:

```sh
# Example: Linux host → Windows x86_64
rustup target add x86_64-pc-windows-gnu
sudo apt install gcc-mingw-w64-x86-64

# Example: Linux host → Linux AArch64
rustup target add aarch64-unknown-linux-gnu
sudo apt install gcc-aarch64-linux-gnu
```

When a required toolchain is not found, `cocotte build` falls back to emitting a **source bundle** — a directory containing a generated `Cargo.toml` and `main.rs` that you can transfer to the target machine and compile there with `cargo build --release`.

### Output naming

| Target           | Output filename                  |
|------------------|----------------------------------|
| Current (no flag)| `<project>`                      |
| linux-x86_64     | `<project>-linux-x86_64`         |
| windows-x86_64   | `<project>-windows-x86_64.exe`   |
| macos-aarch64    | `<project>-macos-aarch64`        |

All binaries land in `dist/` (or the directory specified with `--out`).

## 17. Testing

Test files live in `tests/` and must end with `_test.cot`. They are ordinary `.cot` files; any use of `assert` or `assert_eq` that fails terminates the test run with a descriptive error.

```cocotte
# tests/math_test.cot
library add "libraries/math_utils.cotlib"

assert_eq(math_utils.square(5),  25)
assert_eq(math_utils.square(0),  0)
assert_eq(math_utils.cube(3),    27)
assert_eq(math_utils.is_even(4), true)
assert_eq(math_utils.is_even(7), false)
assert_eq(math_utils.is_prime(2),  true)
assert_eq(math_utils.is_prime(17), true)
assert_eq(math_utils.is_prime(4),  false)

print "All math tests passed."
```

```sh
cocotte test               # run all files in tests/
cocotte test --verbose     # show each assertion
```

`cocotte test` exits 0 if all tests pass, non-zero otherwise — suitable for use in CI.

```cocotte
func word_freq(text)
    var counts = {}
    for word in text.lower().split(" ")
        var w = word.trim().replace(",", "").replace(".", "")
        if not w.is_empty()
            if counts.has_key(w)
                counts.set(w, counts.get(w) + 1)
            else
                counts.set(w, 1)
            end
        end
    end
    return counts
end

var freq = word_freq("the cat sat on the mat the cat")
for word in freq.keys()
```

### Example 5 — CLI note-taking app

```cocotte
module add "json"

var FILE = "notes.json"

func load()
    if not file_exists(FILE)
        return []
    end
    try
        return json.parse(read_file(FILE))
    catch err
        return []
    end
end

func save(notes)
    write_file(FILE, json.stringify(notes))
end

func cmd_add(text)
    var notes = load()
    notes.push({"id": notes.len() + 1, "text": text, "ts": time_now()})
    save(notes)
    print "Saved note #" + notes.len()
end

func cmd_list()
    var notes = load()
    if notes.is_empty()
        print "No notes."
        return nil
    end
    for n in notes
        print "[" + n.get("id") + "] " + n.get("text")
    end
end

func cmd_delete(id)
    var notes = load()
    save(notes.filter(func(n) return n.get("id") != id end))
    print "Deleted #" + id
end

var cmd = input("Command (add/list/delete/quit): ")
while cmd != "quit"
    if cmd == "add"
        cmd_add(input("Text: "))
    elif cmd == "list"
        cmd_list()
    elif cmd == "delete"
        cmd_delete(to_number(input("ID: ")))
    else
        print "Unknown command."
    end
    cmd = input("Command: ")
end
```
